# 🌼 민들레 도매 주문 사이트 — 에이전트 지침서

> **최종 갱신**: 2026-06-04  
> **개발 서버**: `npm run dev` (포트 5173, `--host` LAN 오픈)

이 지침서는 이 프로젝트를 작업하는 AI 에이전트가 전체 구조, 핵심 비즈니스 로직 및 스타일 규칙을 빠르게 파악할 수 있도록 돕습니다. 컨텍스트 축소를 위해 불필요한 이력 정보는 삭제하고 핵심만 요약되어 있습니다.

---

## 1. 기술 스택 및 구조
- **Frontend**: React 19.2 + TypeScript + Vite + Zustand (장바구니 상태 관리)
- **Styling**: 순수 커스텀 CSS (`App.css`, `index.css`) — **Tailwind CSS 사용 금지**
- **Database**: Supabase (PostgreSQL) + RLS(Row Level Security) 설정 완료

---

## 2. 라우팅 및 파일 구조
- `/` (`ProductList`): 전체 상품 목록 (모바일 최적화 3열, 카테고리 필터 및 실시간 상품명 검색 기능 포함)
- `/product/:id` (`ProductDetail`): 상품 상세 (슬라이드쇼, 옵션 수량 선택, 확대 모달)
- `/order` (`OrderPage`): 주문서 작성 (전화번호 자동완성, 카카오 주소 API 연동)
- `/admin` (`AdminPage`): 상품 등록/삭제 및 옵션 CDN(ImgBB API) 연동 대시보드
- `/admin/orders` (`AdminOrdersPage`): 주문 접수 목록 조회 및 상태 변경 대시보드 (주요 어드민 도구)
- `/my-orders` (`MyOrdersPage`): 비회원 거래처 실시간 주문 조회 및 품목별 포장 상태 조회

---

## 3. 핵심 비즈니스 로직

### 3-1. 어드민 인증체계 (이중 검증)
- **어드민 로그인 비밀번호**: `012560`
- 쿠키 `admin_auth = 'true'`(7일 유지)와 Supabase Auth `getSession()` 세션 유효성을 마운트 시 이중 검증합니다.
- 로그인 시 비밀번호 `012560`를 DB의 `admin_auth` 테이블의 bcrypt 해시와 대조하기 위해 `verify_admin_password` RPC 함수를 사용하며, 통과 시 `.env`에 정의된 이메일/비밀번호로 `signInWithPassword()`를 처리하여 RLS를 우회합니다.

### 3-2. 주문 및 미송(이월) 처리 프로세스
- 일반 주문 탭에서 포장 완료 시:
  - **모든 품목 체크 시**: 주문 상태를 `'포장 완료'`로 변경하고 모든 품목의 상태를 `'포장완료'`로 업데이트합니다.
  - **미체크 품목 존재 시**: 주문을 `'포장 완료'`로 처리하되, 체크 품목은 `'포장완료'`, 미체크 품목은 `'미송'` 상태로 변경하며 **새로운 미송 주문서(`misong_orders` 및 `misong_order_items`)를 자동 생성 및 이월**시킵니다.
- 미송 주문 탭에서 미송포장완료 시:
  - 모든 품목 체크 시에만 `'미송포장완료'` 상태로 업데이트할 수 있으며, 동기화 처리되어 원본 `order_items`의 상품도 `'미송포장완료'`로 갱신됩니다.

### 3-3. 당일 영업 주기(새벽 4시 기준) 재주문 시 택배비 면제(합배송) 로직
- 구매자가 주문서 작성 페이지(`/order`)에서 전화번호를 조회하거나 입력할 때, 해당 번호로 현재 영업 주기(한국 표준시 KST 기준 당일 새벽 4시 ~ 다음날 새벽 4시) 내에 접수된 기존 주문서(`orders`)가 존재하면 동일인으로 간주해 합배송 처리를 지원합니다.
- 영업 주기 내 주문 내역이 있는 경우, '택배' 배송 방식을 선택해도 배송비(3,000원)를 자동으로 면제(`0원`) 처리합니다.
- 주문 폼 선택 영역, 결제 예정 금액 요약 카드, 그리고 완료 영수증 화면에 무료 안내 메시지(`0원 (오늘 주문 합배송 무료! 🎁)`)를 동적으로 노출해 인지성을 높였습니다.

### 3-4. 실시간 검증 체크박스 저장 및 상태 롤백
- **체크박스 실시간 연동**: `AdminOrdersPage`의 주문 상품 상세 목록에 있는 체크박스는 클릭 시 즉시 DB(`order_items` 또는 `misong_order_items`)의 `is_checked` 컬럼에 실시간 반영되어, 새로고침하거나 페이지를 이동해도 체크 상태가 영구적으로 보존됩니다.
- **주문 상태 복구(Rollback)**: 
  - 어드민이 주문 상태를 `포장 완료`에서 다시 `주문 확인` 등으로 되돌리면, DB의 모든 품목들이 `item_status = '미포장'`, `status_updated_at = null`, `is_checked = false` 상태로 리셋되고 이월되었던 미송 주문서도 자동 삭제됩니다.
  - 미송 주문을 `미송포장완료`에서 `미송`으로 되돌릴 경우에도 품목들이 `status = '미송'`, `is_checked = false`로 자동 복구됩니다.
- **주문 검색 기능**: 상호명 또는 전화번호(자동으로 하이픈 제거 후 부분 매칭)를 입력하여 해당하는 주문 및 미송 리스트를 즉각적으로 필터링해 표시할 수 있습니다. 검색 결과가 없을 경우 사용자에게 별도 안내 문구를 표시합니다.
- **상품 관리 필터 및 검색**: `AdminPage`의 상품 관리 탭에서 유니크한 카테고리 태그들을 동적으로 불러와 상단에 배치하고, 하단에 상품 ID 또는 상품명으로 검색 가능한 실시간 검색창을 탑재하여 빠른 관리가 가능하도록 하였습니다.
- **구매자용 상품 필터 및 검색**: `ProductList` 페이지 상단에 동적 카테고리 필터 태그와 실시간 상품명 검색 기능을 구축하여 사용자 탐색을 고도화하였습니다.
- **상품 가격 인라인 수정**: `AdminPage`의 상품 관리 탭에서 각 상품의 가격 오른쪽에 `✏️ 수정` 버튼을 제공하여, 별도의 페이지 이동 없이 인라인 입력 폼을 통해 즉시 상품의 도매 가격을 수정하고 Supabase DB에 실시간 반영(업데이트)할 수 있도록 구현하였습니다.

### 3-5. 구매자 주문 취소 기능 및 상태 변경
- 구매자가 주문서 조회 페이지(`/my-orders`)에서 본인의 미확인 또는 주문 확인 단계의 주문에 대해 직접 '주문 취소'를 할 수 있습니다.
- 포장 완료 단계에 도달한 주문에 대해서는 취소가 원천적으로 차단됩니다.
- 주문 취소가 발생하면 DB(`orders`)의 `status` 값이 `'주문 취소'`로 업데이트되며, 실시간으로 구매자 페이지와 어드민 주문 관리 페이지 모두에 반영됩니다.

## 4. 데이터베이스 스키마 요약
모든 테이블은 RLS가 활성화되어 있으며 쓰기 작업은 어드민 권한(`auth.role() = 'authenticated'`)으로 제한됩니다.
- `products`: 상품 정보
- `product_variants`: 옵션(색상) 정보
- `customers`: 거래처 정보 (전화번호 cleaner 적용 필수)
- `orders`: 주문 마스터 (배송/결제/주문상태)
- `order_items`: 주문 상품 상세 (`item_status` 및 `is_checked` 포함)
- `misong_orders` & `misong_order_items`: 이월된 미송 주문서 및 품목 데이터 (`is_checked` 포함)

## 5. 스타일링 및 모바일 가이드
1. **모바일 가독성**: 고령층 거래처가 주 사용층이므로 버튼, 터치 패딩, 텍스트 크기를 항상 크고 시인성 좋게 디자인해야 합니다.
2. **다크/라이트 테마 대비**: 어드민 탭 버튼 등은 어두운/밝은 화면 테마에서 모두 텍스트가 잘 보이도록 대비를 충분히 주어야 합니다. 
   - 액티브 탭: 배경 연한 보라(`rgba(139, 92, 246, 0.15)`), 글자색 검정(`#000000`), 테두리 보라.
   - 인액티브 탭: 배경 연한 회색(`rgba(0, 0, 0, 0.04)`), 글자색 어두운 회색(`#4b5563`).
3. **포장완료 시간 노출**: 모바일 환경에서 화면 레이아웃이 깨지는 현상을 방지하기 위해 포장완료(또는 미송포장완료) 시간은 배송 상태 뱃지 영역이 아닌, 상품 정보 영역 내 옵션 텍스트 하단에 작은 글씨로 출력합니다.
4. **상세보기 및 접기 디자인 통일**: 주문 상세 토글 버튼 및 상품 상세 토글 버튼은 모두 **타원형 캡슐 형태(border-radius: 20px)** 디자인으로 통일하여 일관성을 제공합니다.
5. **카드 토글 인터랙션**: 주문 정보 요약 카드(`order-sheet-summary-bar`) 영역 전체를 클릭할 수 있게 확대하여(`onClick`, `cursor: pointer`), 카드 영역 내 어디를 클릭하든 상세 내용이 토글되도록 동선을 개선하였습니다.
6. **필터 및 검색창 정렬**: 필터/검색 컨트롤 바(`orders-filter-bar`)는 대칭형 대형 레이아웃으로 분리하고, 내부 요소들을 **가운데 정렬(center)** 배치하여 가독성과 사용성을 크게 향상시켰습니다.
7. **구매자 주문서 조회 하단 요약 레이아웃**: 모바일 및 좁은 뷰포트에서 배송 방법, 결제 방법, 총금액 텍스트가 겹치거나 깨지는 문제를 해결하기 위해, 배송 정보는 첫 번째 행에 단독 배치하고 결제 정보와 총 가격은 두 번째 행에 space-between 형태로 배치하는 2행 구조로 레이아웃을 고도화하였습니다.
8. **메인 헤더(민들레 로고 칸) 우측 이동 링크 배치 및 디자인 통일**: 상품 등록/관리 대시보드(`AdminPage`)와 주문 대시보드(`AdminOrdersPage`)의 상호 전환용 네비게이션 링크를 사이트 메인 헤더(`NavigationHeader` in `App.tsx`, 즉 민들레 로고가 있는 칸) 내부 우측의 장바구니 버튼이 나오던 영역에 배치하여, 대시보드 화면에 관계없이 깔끔하고 통합된 레이아웃을 제공합니다. `AdminPage`일 때 메인 헤더 우측에는 `📋 주문 목록 관리 보기 →`로, `AdminOrdersPage`일 때 메인 헤더 우측에는 `← 📂 상품 등록/관리 보기`로 이모지 및 화살표 배치를 통일하였습니다.
9. **접이식 날짜 필터**: `AdminOrdersPage`의 최근 3일, 최근 7일, 최근 30일, 전체보기 등의 기간 필터가 공간을 지나치게 차지하고 화면을 산만하게 만드는 문제를 해결하기 위해, 상호/전화번호 검색창 우측에 `필터 ⚙️` 토글 버튼을 추가하여 필터 버튼 그룹이 접혀있고 필요 시에만 클릭해서 펼쳐보도록 구조를 최적화했습니다.
10. **주문 현황 페이지네이션**: 어드민 주문 리스트의 한 페이지에 무수히 많은 주문 카드가 출력되어 발생하는 스크롤 피로 및 성능 저하를 차단하고자, 클라이언트 사이드 필터링을 거친 플랫 리스트 기준 **한 페이지당 최대 8개**씩 노출되도록 페이지네이션을 구현하고 하단에 이전/다음 이동 컨트롤을 추가했습니다.
11. **상태 관리 드롭다운(Select) 및 상세 토글 바 색상/크기 통일**: 주문서 카드의 `진행 상태:` 텍스트 라벨과 하단 정적 뱃지를 제거하고, `<select>` 드롭다운 선택상자 자체의 색상과 해당 주문 카드의 "주문 상품 상세" 접이식 토글 바의 테두리/배경/글자색이 동일한 상태 색상으로 연동되도록 통일했습니다. 노란색은 한층 선명하고 밝은 톤으로 리팩토링하였습니다. (주문 미확인/미송: 주황 `#f59e0b`, 주문 확인: 밝은 노랑 `#eab308`, 포장 완료/미송포장완료: 초록 `#10b981`). 또한 드롭다운 상자의 크기가 고정되어 문자열 길이에 따라 빈 여백이 많이 발생하지 않도록, 선택된 활성 텍스트 길이에 맞춰 셀렉트 박스 너비(width)가 동적으로 변화(`미송`은 74px, `주문 미확인` 및 `미송포장완료`는 138px, 그 외는 112px)하도록 유동적으로 크기 적응형 스타일을 적용했습니다.
12. **주문 취소 카드 스타일링 및 버튼 배치**:
    - 주문이 취소(`'주문 취소'`)된 경우, 해당 주문 카드 전체의 불투명도를 `opacity: 0.5`로 변경하고, 배경과 테두리를 연한 회색(`rgba(156, 163, 175, 0.03)` 및 `rgba(156, 163, 175, 0.2)`)으로 처리하여 한눈에 취소된 내역임을 알 수 있도록 흐릿하게 표현합니다.
    - 주문 취소 버튼은 주문서 카드 상단 헤더의 레이아웃을 해치지 않도록 카드 하단 요약 영역의 총금액 바로 아래쪽에 우측 정렬된 작고 깔끔한 회색 버튼 형태로 배치하였습니다.
